계산기를 만들어보자!

문제 설명에 따라서 필수 과제 계산기를 만들어 봤다.

STEP 1.  클래스 없이 기본적인 연산을 수행할 수 있는 계산기 만들기
- [ ]  **양의 정수(0 포함)를 입력받기**
- [ ]  **사칙연산 기호(➕,➖,✖️,➗)를 입력받기**
- [ ]  **위에서 입력받은 양의 정수 2개와 사칙연산 기호를 사용하여 연산을 진행한 후 결과값을 출력하기**“
- [ ]  **반복문을 사용하되, 반복의 종료를 알려주는 “exit” 문자열을 입력하기 전까지 무한으로 계산을 진행할 수 있도록 소스 코드를 수정하기**

다음과 같은 조건을 만족하여 정상적으로 출력이 되었으며 처음엔 while문 없이 작성했는데도 정상 작동하여 정상이라고 판단했지만 exit 입력하면 탈출 부분에서 어떤 단어를 입력해도 계산기가 종료되는 것을 보고 반복문이 빠진 것이라고 판단하여 다시 작업해주어 정상 작동하는 것을 확인할 수 있었다.

### STEP 2. 클래스를 적용해 기본적인 연산을 수행할 수 있는 계산기 만들기

- [ ]  사칙연산을 수행 후, 결과값 반환 메서드 구현 & 연산 결과를 저장하는 컬렉션 타입 필드를 가진 Calculator 클래스를 생성
- [ ]  **STEP 1에서 구현한 App 클래스의 main 메서드에 Calculator 클래스가 활용될 수 있도록 수정**
- [ ]  **App 클래스의 main 메서드에서 Calculator 클래스의 연산 결과를 저장하고 있는 컬렉션 필드에 직접 접근하지 못하도록 수정 (캡슐화)**
- [ ]  Calculator 클래스에 저장된 연산 결과들 중 가장 먼저 저장된 데이터를 삭제하는 기능을 가진 메서드를 구현한 후 App 클래스의 main 메서드에 삭제 메서드가 활용될 수 있도록 수정

위의 조건을 만족시키면서 코드를 구성한 결과는 다음과 같다.

사칙연산 기능 수행을 Calculator 클래스로 넘겨 주었고 if와 default 부분에서는 조건에 맞지 않는 경우이므로 return 0;를 이용하여 바로 탈출할 수 있게 해주었다.

외부에서 함부로 손댈 수 없게 private으로 선언한 컬렉션 리스트인 results에 result를 저장하기 위해 results.add(result);를 넣어주었다.

App 클래스로 넘겨줄 getter 메서드와 수정할 수 있게 해줄 setter 메서드를 작성했다. 그 후에는 App 클래스에서 getter로 결과를 확인할 수 있게 출력문을 만들어줬다.

Calculator 클래스에서 결과 값이 0보다 클 경우 첫 번째 인덱스 값(0)을 지울 수 있게 해줬고 App 클래스에서는 문자열 비교를 위해 .equals를 이용하여 “exit”와 동일한 경우 계산기를 종료를 할 수 있게 해줬다.
